Game Tantra

The gaming culture in India is growing Leaps and Bounds now, and has given rise to a whole new industry here - the game development industry. Techfest, in association with Dhruva Interactive, gives you a chance to be on the inside of this whole new exciting field. For the first time ever, a competition where you design the game and others (read judges) play it. What's the fun in it, you ask? Well, winning sure is.

Objective

To implement a small game using "Wild Tangent" as your game engine.

Guidelines

We shall define the basic game play for you. All the elements of the basic game-play should be present in your submission. On top of this, you can add whatever you want to make your game look and behave cool.

Your team size should not exceed 5.

Please read through the specification/guideline document provided at the bottom of this page. We also have a small "introduction to Wild Tangent" section in this page as well as in our document. Also please go through the demo provided at the bottom.

We will provide you with art resources for the basic game play. These are not currently posted on this site yet, once we receive your registration we will post a link to you.

Start

You can start right away. Try to make yourself familiar with the game environment there are lots of resources available on the web.

Once you are done with it, you can contact us for the assets/resources.

End Date

January 20th 2005
All submissions should reach us before 20th January 2005, 5:00 PM IST

Language

Use only C/C++ to implement. (Source code HAS to be provided while submitting)

Development environment

Preferably Microsoft Visual Studio 6.0 or greater. Otherwise you can use anything that you are comfortable with.

About WildTangent

The Web Driver provides a platform for the development of high quality, high performance, compact 2D and 3D content for the Internet. It consists of a high level API for Java, JavaScript, and other COM enabled languages such as C, C++ and Visual Basic, and has a powerful graphical engine underneath. The API provides both 2D and 3D support, allowing the creation of everything from simple 2D or 3D content such as product visualizations for e-commerce, through to full-fledged 3D games. All of these can run within a web page, simply by visiting a web site.

The Web Driver consists of a downloadable component of less than 1MB, and includes an updater mechanism to provide an easy way to distribute new versions with additional functionality and maintenance fixes. The user only has to install the Web Driver once, and after that the updater will automatically update the Web Driver when the computer is connected to the Internet (via a background process that uses idle or unused bandwidth), preventing the need for further delays and downloads at the time the content is viewed. The Updater also provides information to Wild Tangent about error conditions on the user's machine, as well as relevant hardware information necessary to reproduce and fix the issue. With the Updater, we can be extremely proactive about fixing issues and implementing new features, so that we can ensure that the user experience remains a positive one.

The plug-in includes Java and JavaScript interface layers, for use by Java/JavaScript enabled Internet Explorer and Netscape browsers. While the Web Driver was designed for the Internet, it also has a COM interface that can be used in standalone applications as well, using languages such as Visual Basic, C, and C++.

There is extensive documentation available in the Developer Central section of our website on how to develop and use the Web Driver. This document provides information on the design goals of the Web Driver, and how those goals are met through the Web Driver's features.

http://www.wildtangent.com/developer/downloads/overviews/WebDriverOverview/index.html

Submission:

  • Submission should be made on a CD.
  • All submissions should reach us before 20th January 2005, 5:00 PM IST.
    • Please courier the submission CD.
    • Make sure that you dispatch it early, so that it reaches us on time.
    • Late submissions will be disqualified.
  • Submission should follow this folder structure.
  • The mandatory folders are explained below:
    • Binary --- Contains the executable and other resources that are required to run your game.
    • Graphics -- Contains any extra graphic resources that you may have used (other than the ones provided). This folder can be empty also.
    • Source Code - Contains all the source code that you have written. It should also include the project, and other necessary files.
  • The game should be able to run from the CD itself.
    • The game should not have any installer. Should be in its original executable form.
    • Make sure you check this before you dispatch it.
    • If it fails, your submission will be disqualified.
  • There should be a "readme.txt" or "readme.doc" file in the "Your team name" folder
    • This file should contain the following information:
      • All your team members name & e-mail address
      • Postal address (of each team member)
      • College name & postal address (of each team member)
      • Place of origin (of each team member)
      • Male/Female (of each team member)
      • Date-of-birth (of each team member)
      • Any additional info regarding how to run the game, etc.
      • Please mention and highlight the USP (Unique selling point) of your game
      • Also, highlight what features you have added on your own.
  • The neatness of submission carries a few points while evaluating.
  • Write the name of your team, college, and place on the CD (with a soft-tip pen).
    • Writing should be legible.
